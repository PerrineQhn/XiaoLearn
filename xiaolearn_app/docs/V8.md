# V8 — Enrichissement B2.1 (exercices, vocab, textes longs)

**Contexte :** suite de V7 (révision multi-leçons, flashcards V3, bilans). V8 s'attaque au vrai goulot d'étranglement du parcours CECR : il n'y avait pas assez de **densité** au-delà de B1.2 pour que l'apprenant atteigne effectivement le niveau annoncé. V8 corrige ça de trois façons complémentaires et livre la **passe complète B2.1** — priorité définie avec NoComment (« B2.1 + B2.2 en priorité, démarre par B2.1 »).

`npx tsc --noEmit` : **0 erreurs** après chaque étape.

## 1. Schéma d'exercice enrichi (6 types + catégorie)

Fichier : `src/pages/StructuredLessonPageV2.tsx`

- `LessonV2ExerciseType` passe de `'mcq' | 'fill'` à :
  `'mcq' | 'fill' | 'order' | 'grammar-quiz' | 'translation' | 'error-correction'`.
- Nouveau champ `category?: 'vocabulary' | 'grammar' | 'translation' | 'listening' | 'reading'` sur `LessonV2Exercise` : permet de différencier un exo grammaire d'un exo vocabulaire dans la même leçon (demande explicite du user).
- Composant dédié `OrderExerciseCard` pour la remise dans l'ordre :
  - seed Fisher-Yates sur `exercise.id` → ordre stable entre re-renders ;
  - `exercise.choices` contient les segments **dans l'ordre correct** (l'UI les mélange) ;
  - validation via `onSelect(isCorrect ? correctIndex : -1)`, compatible avec le state machine existant ;
  - feedback affiche ce que l'utilisateur a construit vs la bonne réponse.
- `getExerciseBadge()` → badge pédagogique coloré (gram-quiz, translation, error-correction, order).

Styles : `src/styles/lesson-v2.css` (badges `.lv2-exercise-badge--*`, chips `.lv2-order-chip--picked/pool`).
