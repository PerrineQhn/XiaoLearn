## 4. Architecture IA du simulateur

```
┌─────────────────────┐       ┌──────────────────────────┐
│  SimulatorScenario  │──────▶│ buildSimulatorSystemPrompt│──┐
└─────────────────────┘       └──────────────────────────┘  │
                                                            ▼
                              ┌──────────────────────────┐
priorTurns + userMessage ────▶│   sendSimulatorMessage   │─▶ Gemini 2.5 Flash
                              └──────────────────────────┘
                                            │
                                            ▼
                              ┌──────────────────────────┐
                              │ parseSimulatorResponse   │
                              │  [hanzi]                 │
                              │  [pinyin]                │
                              │  *[fr italic]*           │
                              │  [SCENARIO_COMPLETE]?    │
                              └──────────────────────────┘
```

- Réutilise `generateGeminiResponse` de `services/geminiService.ts` — pas de
  second fournisseur à configurer.
- Le system prompt contient : persona, objectif, niveau CECR, hint de
  registre, vocabulaire du scénario, étapes numérotées avec critères de
  complétion, 6 règles strictes dont la règle de recast implicite.
- `[SCENARIO_COMPLETE]` est dépouillé à la lecture : il sert de signal
  d'arrivée de l'objectif, pas de texte affiché à l'utilisateur.

## 7. Cadence SRS

`useSrsPreferences` persiste la cadence dans `localStorage`
(`cl_srs_preferences_v1`) et propose 4 presets :

| Preset | Cartes / jour | Usage recommandé |
| ------ | ------------- | ---------------- |
| Tranquille | 10 | Routine quotidienne sans pression |
| Standard | 20 | Cadence Anki recommandée (~30 min/jour) |
| Intensif | 30 | Progression rapide (~45 min/jour) |
| Marathon | 50 | Sprint pré-examen (temporaire) |

Un mode `custom` accepte n'importe quelle valeur entre 1 et 200.

**Note de câblage** : la valeur est déjà lue par défaut par
`useFlashcardSRS` (via sa constante `DAILY_NEW_CARDS = 20`). Pour qu'une
page consomme dynamiquement la préférence utilisateur, il suffit d'appeler
`useSrsPreferences().preferences.dailyNewCards` et de le passer en
`options.dailyNewCards`. Les pages SRS V2 à venir devront s'y brancher.

## 9. Audit authenticité chinoise (V6-10)

Passe de relecture sur les 20 `openingLineHanzi` + vocab pour s'assurer que
les formulations sonnent comme un natif, pas comme une traduction calquée du
français. 6 corrections appliquées :

| Scénario | Avant | Après | Raison |
| -------- | ----- | ----- | ------ |
| `sim-cafe` | `热的 → 're de'` | `'rè de'` | Marques de tons manquantes (bug) |
| `sim-cafe` | `你好，请问你要什么？` | `您好，请问需要点什么？` | Un barista vouvoie + "需要点什么" est l'idiome café standard |
| `sim-taxi` | `师傅，您去哪儿？` | `您好，咱们去哪儿？` | 师傅 est l'appellatif utilisé PAR le passager VERS le chauffeur, pas l'inverse. 咱们 = registre chaleureux typiquement pékinois |
| `sim-pharmacy` | `你好，你哪里不舒服？` | `您好，您哪儿不舒服？` | Pharmacien vouvoie ; 哪儿 plus oral que 哪里 |
| `sim-hospital` | `请坐，你哪里不舒服？` | `请坐，您哪儿不舒服？` | Même raison que pharmacie |
| `sim-police` | `请冷静一点，您要报什么警？` | `请坐，慢慢说，发生什么事了？` | "报什么警" est directif/maladroit ; un agent d'accueil dirait d'abord "asseyez-vous, racontez" |

Éléments validés sans modification (prouvés idiomatiques) :
- `欢迎光临！几位？` (restaurant) — scripte standard
- `师傅，您去哪儿？` renversé vers `您好，咱们去哪儿？` ci-dessus
- `来，大家先端起杯子，我先敬大家一杯！` (饭局) — formule native, avec piège
  culturel intégré (jeu sur la hauteur des verres)
- `快进来，快进来！路上辛苦了吧？` (belle-famille) — chaleur familiale réelle
- `哼，你昨天怎么放我鸽子啊？` (excuses) — idiome natif 放鸽子 bien utilisé
- Les 20 `successMessageFr` mixent FR + emoji — délibéré, ton XiaoLearn.

Sanity check post-correction : `npx tsc --noEmit` → 0 erreur ; 20/20 scénarios
valides, 20/20 vocab items avec pinyin correct.

## 11. Ce qui n'est volontairement pas fait

- **Pas de rewiring de FlashcardPageV2** vers la préférence SRS : la page
  V2 actuelle est une vue « collection / liste », pas un runner SRS. Un
  futur SrsSessionPageV2 pourra consommer `useSrsPreferences` directement.
- **Pas d'UI dans HomePageV2** pour le simulateur : l'onglet principal
  suffit (demande explicite). Peut être ajouté via une card shortcut dans
  une itération ultérieure.
- **Pas de tagging systématique** des flashcards existantes par registre.
  Le champ `register` est optionnel sur `SimulatorVocabItem` ; l'étendre
  à `LessonItem` demanderait un tri manuel sur 11k+ entrées — hors
  périmètre V6.
- **Pas de détection LLM-driven** de l'étape en cours : la progression est
  linéaire (heuristique `guessCurrentStepIndex`). Un vrai routeur d'étape
  piloté par le LLM (parse JSON structuré) peut être greffé plus tard sans
  changer le schéma.
