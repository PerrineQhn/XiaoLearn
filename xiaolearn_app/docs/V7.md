# XiaoLearn V7 — Révision multi-leçons, Flashcards V3, Bilans de niveau

Cette note documente l'initiative V7, livrée en une passe. Elle couvre trois
axes inspirés du dashboard Seonsaengnim : révision SRS au niveau **leçon**
(et non plus seulement au niveau flashcard), upgrade des flashcards (mode
inverse, deck personnel, deck de phrases), et bilans de fin de niveau CECR
avec bannière dans le parcours.

---

Questions piochées dans `cecrExercisesV2`. Écran de résultats : mastery avant / après, prochaine date, liste des erreurs avec « refaire la leçon ». |

## 4. Révision multi-leçons (ReviewPageV3)

### 4.1 Mastery pondérée

`useLessonMastery` stocke pour chaque `lessonId` :

```ts
interface LessonMastery {
  lessonId: string;
  level: CecrLevelSlug;
  scores: number[];           // 5 derniers scores (0-100), plus récent en fin
  masteryPct: number;         // moyenne pondérée weights = [3, 2.5, 2, 1.5, 1]
  srsLevel: number;           // 0..5 (indexe LESSON_SRS_INTERVALS_DAYS)
  nextReviewAt: string;       // ISO date
  lastReviewedAt: string;     // ISO date
  reviewCount: number;
}
```

Le poids rolling donne plus d'importance au dernier essai → le score remonte
vite quand l'élève se débloque, et retombe rapidement en cas de régression.

### 4.2 SRS leçon-level

```ts
const LESSON_SRS_INTERVALS_DAYS = [1, 3, 7, 15, 30, 60];
```

- `recordSessionResult(lessonId, level, scorePct)` :
  - échec (`scorePct < 60`) → `srsLevel = 0`, prochaine review demain
  - moyen (`60 ≤ scorePct < 80`) → palier conservé, prochaine review à +J
  - réussite (`scorePct ≥ 80`) → palier +1 (cap 5), prochaine review à +J

### 4.3 Modes

| Mode | Source | Volume |
| ---- | ------ | ------ |
| `smart-mix` | 50 % due + 30 % faiblesses + 20 % nouvelles/aléatoires | 3 / 6 / 10 |
| `due-today` | leçons dont `nextReviewAt ≤ today` | 3 / 6 / 10 |
| `weaknesses` | leçons avec `masteryPct < 60` | 3 / 6 / 10 |
| `free` | pick libre parmi toutes les leçons complétées | 3 / 6 / 10 |

Les questions sont tirées de `cecrExercisesV2` agrégés par `buildLessonIndex`
(dans `ReviewPageV3`) pour garantir un pool suffisant.

### 4.4 Scorecard

En fin de session :

- Mastery **avant / après** par leçon + delta
- Prochaine date de révision (calculée post-session)
- Liste des erreurs avec CTA « Refaire la leçon » (redirige vers la leçon)
- XP : `REVIEW_XP_PER_CORRECT = 3` × bonnes réponses + `REVIEW_XP_SESSION_BONUS = 15` si session ≥ 80 %

## 5. Flashcards V3 (FlashcardPageV3)

### 5.1 Direction pill

Composant pill-switch dans le header, persistant en `localStorage`:

- `hanzi→fr` (défaut) : face avant = hanzi + pinyin, face arrière = traduction
- `fr→hanzi` : face avant = traduction, face arrière = hanzi + pinyin (pinyin masqué par des points dans la liste pour forcer le rappel actif)

### 5.2 Deck personnel

- CRUD complet via `usePersonalFlashcards`
- Modal `PersonalCardModal` avec : `hanzi`, `pinyin` (auto si vide, via `lookupPinyinForHanzi` → `pinyin-pro`), `translationFr`, `translationEn`, `note`
- Quota : `PERSONAL_FLASHCARDS_MAX = 500`, barre gold → rouge chinois pour indiquer la saturation
- SRS identique aux flashcards cœur (intervals [1, 3, 7, 14, 30, 60, 120])
- Dédoublonnage sur `hanzi + translation` pour éviter les conflits avec les flashcards cœur

### 5.3 Onglet « phrases »

- Source : `dialogue.lines[]` des `LessonPath.modules` des leçons complétées
- Mapping `{lessonId, speaker, hanzi, pinyin, translationFr}` → `SentenceFlashcard`
- ID déterministe : `sent-{lessonId}-{idx}` (permet de persister le SRS indépendamment du deck words)
- Bouton « ouvrir le dialogue » pour retourner au dialogue source depuis la phrase

## 6. Bilans de fin de niveau (LevelBilanPage)

### 6.1 Bannière d'entrée

`LevelBilanBanner` est rendu via le prop `renderLevelBanner` de
`LessonPathsPage`. Les états sont :

| État | Condition | Rendu |
| ---- | --------- | ----- |
| `locked` | `completionPct < 80` | Barre de progression « X / Y leçons » |
| `available` | `completionPct ≥ 80` + pas encore passé | CTA « 🏆 Lancer le bilan » |
| `passed` | `firstPassAt` renseigné | Insigne or + bestScore + CTA « rejouer » |
| `legacy-recognized` | `markLegacyRecognized` appelé | « Parcours antérieur reconnu » (sans XP) |

### 6.2 Questions & scoring

- Fichier `src/data/cecr-bilans.ts` : 10 bilans × 10 questions manuscrites
- Types : `mcq` (4 choix) uniquement en V7, pour simplicité
- Seuil `BILAN_DEFAULT_PASSING = 0.8` (80 %)
- `BILAN_XP_REWARD = 60` au **premier** passage validé (idempotent : le hook ne déclenche `onFirstPass` qu'une fois par niveau)
- Retry illimité : les tentatives postérieures mettent à jour `bestScore` sans XP supplémentaire

### 6.3 Reconnaissance legacy

Pour les utilisateurs qui avaient déjà passé du contenu équivalent en
structure pré-CECR (HSK), `LevelBilanPage` expose un CTA secondaire
« Mon parcours a déjà été validé ». Action : `levelBilans.markLegacyRecognized(level)`
→ l'état bascule en `legacy-recognized`, sans XP, et débloque la bannière
d'entrée du niveau suivant si la hiérarchie le requiert.

## 7. Hooks — API récapitulative

### `useLessonMastery({ syncEnabled })`

```ts
interface UseLessonMasteryAPI {
  recordSessionResult(lessonId: string, level: CecrLevelSlug, scorePct: number): LessonMastery;
  seedLessonMastery(lessonId: string, level: CecrLevelSlug): void; // appelé quand une leçon est complétée pour la 1ère fois
  getMastery(lessonId: string): LessonMastery | undefined;
  getLevelMastery(level: CecrLevelSlug): LessonMastery[];
  getDueLessons(today?: string): LessonMastery[];
  getWeaknesses(threshold?: number): LessonMastery[];
  allMastery(): LessonMastery[];

```

```ts
interface UseLevelBilansAPI {
  recordAttempt(level: CecrLevelSlug, scorePct: number, passingThreshold?: number): LevelBilanEntry;
  markLegacyRecognized(level: CecrLevelSlug): void;
  getEntry(level: CecrLevelSlug | null): LevelBilanEntry | undefined;
  hasPassed(level: CecrLevelSlug): boolean;
  allEntries(): LevelBilanEntry[];

```

interface UsePersonalFlashcardsAPI {

  deleteCard(id: string): void;
  recordReview(id: string, quality: 1 | 2 | 3 | 4): void;

## 12. Inspirations

Les trois axes V7 s'appuient sur trois pages clés du dashboard Seonsaengnim
(`/dashboard/reviews`, `/dashboard/flashcards`, `/dashboard/placement`), déjà
documentées dans la note d'inspiration. Les patterns réutilisés :

- **Barre mastery pondérée** (review-v3) — progression visuelle par leçon, pas par item
- **Direction pill + chips** (flashcards-v3) — commutation mode de rappel en 1 clic
- **Scorecard bilingue** (level-bilan) — score + erreurs + CTA « refaire »
- **Bannière contextuelle** (lesson-paths) — déblocage visible dans le flux naturel
